# Generating a CSR without macOS

If you don’t have a Mac, you can still create the Certificate Signing Request (CSR) from Windows or Linux using OpenSSL (available in Git Bash, WSL, or any environment with OpenSSL installed).

## Steps

1. **Install OpenSSL**
   - Git for Windows already bundles OpenSSL in Git Bash.
   - On Linux/WSL, install via your package manager (`sudo apt install openssl`).

2. **Generate a private key**
   ```bash
   openssl genrsa -out parent_app.key 2048
   ```

3. **Create the CSR**
   ```bash
   openssl req -new -key parent_app.key -out parent_app.csr
   ```
   OpenSSL prompts for:
   - `Country Name`, `State`, etc. (can be filled with organization details).
   - `Common Name`: use your Apple Developer email.
   - Leave `challenge password` blank.

4. **Upload CSR**
   - Go to Apple Developer → Certificates → “+” → choose certificate type.
   - Upload `parent_app.csr`.

5. **Install & export certificate**
   - After Apple issues the `.cer`, download it.
   - Convert `.cer` + key into `.p12` on Windows/Linux:
     ```bash
     openssl x509 -in downloaded.cer -inform DER -out parent_app.pem -outform PEM
     openssl pkcs12 -export -inkey parent_app.key -in parent_app.pem -out parent_app.p12
     ```
   - When prompted, set a password for the `.p12` (store it for GitHub secrets).

6. **Cleanup**
   - Keep `parent_app.p12` and `parent_app.key` in secure storage.
   - You only need the `.p12` for CI; base64-encode it before adding to GitHub secrets:
     ```bash
     base64 parent_app.p12 > parent_app.p12.b64
     ```

This CSR works exactly like one generated by Keychain Access, so you can complete Apple provisioning entirely from Git Bash/WSL.
